Manipulando o DOM

Document Object Model - Criado pelo W3C, O DOM é uma multi-plataforma que representa como as marcações em HTML, XHTML e XML são organizadas e lidas pelo navegador que você usa. Uma vez indexadas, estas marcações se transformam em elementos de uma árvore que você pode manipular via API – que é o que fazemos quando usamos programas ou scripts para alterar funcionalidades de uma página: conteúdo, estrutura ou folha de estilo.

  • fonte: https://tableless.com.br/tenha-o-dom/
// pegar um elemento
  const app = document.getElementById('app');
  // criar um elemento
  const spanHello = document.createElement('span');

  // adicionar atributos
  spanHello.className = 'my-span-class';
  spanHello.id = 'my-span-id';
  // criar um nó de texto
  const text = document.createTextNode('Hello');
  // adicionar filhos
  spanHello.appendChild(text);
  app.appendChild(span);

  const myLink = document.createElement('a');
  myLink.setAttribute('href', 'http://google.com');

  app.appendChild(myLink);

  // remover filho
  app.removeChild(myLink);

Funções

As funções em Javascript são First Class Functions, ou seja, funções podem ser passadas como argumentos para outras funções, podem ser retornadas de outras funções, podem ser atribuidas a variáveis e serem armazenadas em qualquer estrutura de dados como arrays e objetos. Meio confuso? Não se preocupe, com o tempo se tornar mais claro.

Abaixo segue algumas formas de invocar funções.

function sayHello(name) {
  return 'Hello, ' + name;
}

sayHello('John');

var sayHello = function(name) {
  return 'Hello, ' + name;
}
// atribui uma função a variavel, e a mesma retorna outra função
var makeHello = function(name) {
  return function() {
    return 'Hello, ' + name;
  }
}

var helloJohn = makeHello('John');
var helloMark = makeHello('Mark');

helloJohn();
helloMark();

Callbacks

Callback é um padrão (pattern) onde uma função recebe outra função que será chamada postetiormente em um momento apropriado. Essencialmente criado par alidar com a natureza assíncrona do Javascript.

Você irá ver esse padrão em todo lugar quando começar a usar Javascript.

var shouldCall = true;

var myCallback = function() {
  console.log('Serei chamado quando a outra função quiser :P');
}

function printNames(shouldCall, callback) {
  if (shouldCall) {
    callback();
  }
  console.log(':P');
}

printNames(shouldCall, myCallback);

Closure

Closures são funções que se referem a variáveis livres (independentes). Em outras palavras, a função definida no closure "lembra" do ambiente em que ela foi criada.

function sayHello() {
  var say = function() { console.log(hello); }
  var hello = 'Hello, world!';
  return say;
}
var sayHelloClosure = sayHello();
sayHelloClosure(); // ‘Hello, world!’

// Outro exemplo
var x = 10;
function foo() {
  var y = 20; // free variable
  function bar() {
    var z = 15; // free variable
    return x + y + z;
  }
  return bar;
}

Novas Features

O Javascript foi criado através das especificações da ECMAScript, várias alterações/inclusões na sintaxe da linguagem ocorreram, portanto vou colocar abaixo algumas que iremos usar no nosso curso.

// const, let
var name = 'palmer';
var age = 35;
age = 36;

// novo
const name = 'Palmer';
let age = 35;
age = 36;

// Template strings
const name = 'Roberbo';
const myName = 'My name is ' + name;

// novo
const myName = `My name is ${name}`;

// arrow functions
function hello(name) {
  return 'my name is ' + name;
}

// novo
const hello = name => `my name is ${name}`;

// Destructuring assignment
const person = { name: 'Matias', age: 34, points: 450 };
const name = person.name;
const age = person.age;
const points = person.points;

// novo
const { name, age, points } = person;

Event Listeners

addEventListener() registra uma única espera de evento em um único alvo. O alvo do evento pode ser um único elemento em um documento, o documento em si, uma janela, ou um XMLHttpRequest.

Para registrar mais de uma espera de evento como alvo, chame addEventListener() para o mesmo alvo mas com diferentes tipos de evento ou captura de parâmetros.

const input = document.getElementById('myInput');

  input.addEventListener('keyup', event => {
    console.log(event.target);
    console.log(event.target.value);
  });

  const button = document.querySelector('#myButton');

  button.addEventListener('click', event => {
    console.log(event.target);
    console.log(event.target.value);
  });

Foreach, Map, Filter e Reduce

Essas funções são extremamente usadas, portando dê uma atenção.

Recomendo de fato entrar no blog e dar uma boa olhada e fazendo seus próprios testes. Reduce é uma das funções mais ricas, mas também um pouco mais difícil de entender, mas não tema!

const data = [
  { name: 'Galadriel', race: 'Elves', weapons: [ 'Elven Magic', 'Nenya' ] },
  { name: 'Legolas', race: 'Elves', weapons: [ 'Bow', 'Knife' ] },
  { name: 'Gandalf', race: 'Maiar', weapons: [ 'Glamdring', 'Wizard Staff', 'Sword' ] },
  { name: 'Radagast', race: 'Maiar', weapons: [ 'Powers of the Maiar', 'Wizard Staff' ] },
  { name: 'Aragorn', race: 'Men', weapons: [ 'Anduril', 'Sword' ] },
  { name: 'Sauron', race: 'Ainur', weapons: [ 'One Ring', 'Sword', 'Powers of the Maiar' ] },
  { name: 'Faramir', race: 'Men', weapons: [ 'Bow', 'Sword' ] }
];

// mostrar os nomes
data.forEach(item => {
  console.log(item.name);
});

const getNames = data.map(item => item.name);
console.log(getNames);

const filterOnlyTheElves = data.filter(item => item.race === 'Elves');
console.log(filterOnlyTheElves);

const countTheWeapons = data.reduce((acc, item) => {
  acc += item.weapons.length;
  return acc;
}, 0);

console.log(countTheWeapons);
